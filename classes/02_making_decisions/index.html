Multi-Armed Bandits

Theme: Making decisions under uncertainty (clinical trials example)

Based on the ideas from Reinforcement Learning: An Introduction (Sutton & Barto), pp. 25–36.

Agenda

  • Bandit problem setup
  • Clinical trials as a decision problem
  • Action-value definition and expected reward
  • Estimating action values
  • \(\varepsilon\) - greedy action selection

The Bandit Problem

You have \(k\) actions (arms). Each action \(a\) produces a reward \(R\) drawn from an unknown distribution.

Goal: maximize cumulative reward by balancing exploration and exploitation.

Clinical Trials Scenario

Imagine \(k\) treatments for the same condition.

  • Each patient is a trial
  • A treatment is an action
  • Observed outcome is a reward
  • We must learn which treatment works best

Notation

  • \(A_t\): action chosen at time \(t\)
  • \(R_t\): reward observed after choosing \(A_t\)
  • \(q_*(a)\): true value of action \(a\)

Expected Reward of an Action

The true action value is the expected reward of action \(a\):

\[ q_*(a) = \mathbb{E}[R_t \mid A_t = a] \]

We do not know \(q_*(a)\), so we must estimate it from data.

Action-Value Estimate (Sample Average)

Let \(N_t(a)\) be the number of times action \(a\) has been selected up to time \(t\).

\[ \hat{q}_t(a) = \frac{1}{N_t(a)} \sum_{i=1}^{t} R_i \cdot \mathbb{1}\{A_i = a\} \]

This estimate converges to \(q_*(a)\) as \(N_t(a)\to\infty\).

Incremental Update Form

Instead of storing all rewards, update online:

\[ \hat{q}_{t+1}(A_t) = \hat{q}_t(A_t) + \frac{1}{N_t(A_t)}\left(R_t - \hat{q}_t(A_t)\right) \]

Only the selected action is updated each step.

Explore vs. Exploit

  • Exploit: choose the best estimated action
  • Explore: try actions with uncertain or lower estimates

Clinical trials must explore to discover better treatments, but also exploit to help current patients.

\(\varepsilon\) - Greedy Action Selection

Choose:

  • with probability \(\varepsilon\), a random action (explore)
  • with probability \(1-\varepsilon\), the greedy action (exploit)
\[ A_t = \begin{cases} ext{random action}, & \text{with prob. } \varepsilon \\ \arg\max_a \hat{q}_t(a), & \text{with prob. } 1-\varepsilon \end{cases} \]

What Does \(\varepsilon\) Control?

  • Larger \(\varepsilon\) $ ightarrow$ more exploration
  • Smaller \(\varepsilon\) $ ightarrow$ more exploitation

A common choice is \(\varepsilon = 0.1\), but it depends on the problem.

Clinical Trials: Why \(\varepsilon\) - Greedy?

  • Avoids prematurely committing to a suboptimal treatment
  • Ensures all treatments are sampled
  • Simple and effective baseline method

Summary

  • Multi-armed bandits model decision-making with uncertainty
  • Action value: \(q_*(a) = \mathbb{E}[R_t \mid A_t=a]\)
  • Estimate values via sample averages or incremental updates
  • \(\varepsilon\)-greedy balances exploration and exploitation
